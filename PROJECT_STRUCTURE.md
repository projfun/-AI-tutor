```

```

### Расписание
```

🌐 UI: web/index.html (schedule-card)
🔄 Обновление: main.py (loadSchedule)
```

### Олламе
```
⚙️ Проверка: env_setup.py (is_ollama_installed)
📥 Установка: env_setup.py (download_ollama)
🚀 Запуск: env_setup.py (start_ollama)
🔌 API: engine.py (OLLAMA_API)
```

```
📖 Загрузка: rag.py (add_textbook)
💾 БД: data/chroma_db/ (ChromaDB)
```

## 🔧 Технические решения

### Почему Ollama?
- ✅ Полностью локально (приватность)
- ✅ Работает на любом ПК
- ✅ Легко качать модели
- ✅ Хорошая производительность
- ✅ Открытый исходный код

### Почему Llama 3.1?
- ✅ Мощная и проверенная модель
- ✅ Хорошее качество ответов ученикам
- ✅ Квантованная версия (q4_K_M) весит 4.5 ГБ
- ✅ MIT лицензия (можно использовать)

### Почему DuckDuckGo?
- ✅ Не нужен API ключ
- ✅ Бесплатно и анонимно
- ✅ Блокирует трекинг
- ✅ Достаточное качество результатов

### Почему ChromaDB?
- ✅ Простой в использовании
- ✅ Встроенная векторизация
- ✅ Работает локально
- ✅ Быстро для небольших БД

### Почему Eel?
- ✅ Python бэкенд + Веб UI в одном приложении
- ✅ Не нужен Node.js
- ✅ Простая синхронизация Python ↔ JavaScript
- ✅ Красивый и отзывчивый интерфейс
