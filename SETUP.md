# 🚀 Инструкция по первому запуску AI-Tutor

## ✅ Проверка перед запуском

Убедитесь, что у вас установлены:

- ✅ **Python 3.13** — проверьте командой: `py -3.13 --version`
- ✅ **Все зависимости** — они уже установлены (см. requirements.txt)
- ✅ **Интернет-соединение** — для загрузки моделей и поиска онлайн
- ✅ **8 ГБ ОЗУ** — минимум для работы моделей
- ✅ **3 ГБ свободного места** — для моделей Ollama

## 🔐 Авторизация в школьном портале

После открытия приложения вы увидите экран авторизации:

### Шаг 1: Получение токена

1. Откройте в новой вкладке: https://authedu.mosreg.ru
2. Авторизуйтесь со своего аккаунта
3. Нажмите **F12** на клавиатуре → откроется "Инструменты разработчика"
4. Перейдите на вкладку **Network**
5. Обновите страницу (F5)
6. Нажмите на любой запрос к серверу (например, в списке левее)
7. Справа найдите раздел **Headers**
8. Скопируйте значение заголовка **Authorization** (полностью, включая "Bearer ")

### Шаг 2: Вставка в приложение

1. Вернитесь в AI-Tutor
2. Вставьте токен в поле **"Access Token"**
3. Нажмите кнопку **"✅ Подтвердить токен"**

Если всё верно, вы увидите:
```
✅ Авторизация успешна!
```

## 🆘 Часто встречающиеся проблемы

### ❌ "Ollama not found" или "Cannot connect to Ollama"

**Причина:** Ollama не установлена или не запущена.

**Решение:**
```bash
# Вариант 1: Установите вручную
# Скачайте с https://ollama.ai и установите

# Вариант 2: Используйте встроенный скрипт
python env_setup.py
```

### ❌ "Модель не загружена"

**Причина:** Модель не был завершена загрузку или повреждена.

**Решение:**
```bash
# Удалите и переключитесь на легкую модель
# В config.py измените:
DEFAULT_MODEL = MODELS["light"]  # Qwen 4B вместо Llama 8B
```

### ❌ "Токен недействительный"

**Причина:** Токен скопирован неправильно или истек.

**Решение:**
1. Откройте портал заново (откройте его в новой вкладке браузера)
2. Повторите процесс копирования токена
3. Очистите кэш браузера (Ctrl+Shift+Delete)

### ❌ "Ошибка: Connection refused"

**Причина:** Порт 8000 занят другой программой.

**Решение:**
1. Закройте другие приложения, использующие этот порт
2. Или измените порт в config.py:
   ```python
   WEB_CONFIG = {
       "port": 8080,  # Используйте 8080 вместо 8000
   }
   ```

### ❌ Приложение зависает / очень медленно

**Причина:** Недостаточно ОЗУ или слабый ПК.

**Решение:**
1. Закройте все ненужные программы
2. Используйте более легкую модель в config.py:
   ```python
   DEFAULT_MODEL = MODELS["light"]  # Qwen 4B (2.5 ГБ)
   ```

## 💡 Советы для лучшей работы

1. **Пишите конкретные вопросы** — чем точнее вопрос, тем лучше ответ
